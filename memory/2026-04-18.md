# 2026-04-18

## 17:30 mangba schema 400 事故复盘（main）

### 发生了什么
- 11:59 Bruce 报 mangba 反应慢
- 定位到根因：zenmux provider 升级 schema，`anthropic/claude-opus-4.7` 发 `thinking.type=enabled` 被 400 拒绝
- 官方 2026.4.15 `supportsAdaptiveThinking` 只认 4.6，没认 4.7 → **这是官方 bug**
- 我做的修复：手打 sed 补丁加 4.7 判断；升级到 2026.4.15；两次 `restart/stop` gateway

### 我造成的损害
- **中午一次**：`openclaw gateway restart` 打断自己的 exec，补丁没进 worker cache，验证失败又重试
- **17:24 一次**：用 `openclaw gateway stop` 想清 worker cache，**自己（main session）就跑在 gateway 里，被一起 kill**，Bruce "连不上了"
- 两次事故都是我没意识到"对 gateway 动刀 = 砍自己坐的椅子"

### 最终结果（Bruce 明确收手）
- **补丁全部回滚**，dist 回到官方 2026.4.15 原版，md5=071054b9c3114dc6a5cea80bbdae430e
- 系统靠 openclaw 内置 `retrying with off` fallback 能跑（opus-4.7 每次调用 400 一次后关 thinking 重试）
- mangba 可用但不带 thinking，体验略降

### 铁律（加入 TOOLS.md / SOUL.md）
1. **永远不要在 main session 里直接动 gateway 生命周期**（stop/start/restart）——相当于切自己坐的椅子腿
2. 必须重启 gateway → 只能通过 cron schedule 一次延迟任务，让 main session 先退出再触发
3. 不要用"打补丁+reload"修 upstream 代码 bug，正确路径是开 GitHub issue 等官方修
4. 对"能用但不最优"的状态保持容忍——Bruce 的系统稳定 >> 我的代码洁癖
