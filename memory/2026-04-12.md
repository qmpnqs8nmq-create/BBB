## 17:20 修复 webchat 图片上传功能

### 问题
Bruce 上传图片后收不到，Gateway 日志报 `parseMessageWithAttachments: attachment(s) dropped — model does not support images`

### 根因
2026.4.10 新增 bundled `codex` provider（#64298），通过 implicit discovery 往 models.json 写入无 apiKey 的模型定义 → pi-sdk validateConfig 失败 → 所有 zenmux 自定义模型从 catalog 消失 → Gateway 判定当前模型不支持图片 → 附件被丢弃。

### 修复
1. `openclaw.json` 中 `.models.providers.openai.models` 改为 `[]`
2. 新增 `.models.providers.codex` 带 `apiKey: "oauth-managed"` 占位符
3. 配置 `imageModel: openai-codex/gpt-5.4`、`tools.media.audio` 转录模型
4. 全部 9 个 agent 的 models.json 同步修复
5. Gateway 重启后图片功能恢复

### 教训
- models.json 是生成文件，修复必须改 openclaw.json
- 规则 3 违反：前几次改了 models.json 导致反复试错
- 这是 OpenClaw regression bug，应提 issue
