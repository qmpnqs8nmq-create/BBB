
## 17:20 修复 webchat 图片上传功能

### 问题
Bruce 上传图片后我收不到，Gateway 日志报 `parseMessageWithAttachments: attachment(s) dropped — model does not support images`

### 根因
2026.4.10 新增 bundled `codex` provider（#64298），它通过 implicit discovery 往 models.json 写入了没有 apiKey 的模型定义。pi-sdk 的 `validateConfig` 要求自定义模型必须有 apiKey，codex（和之前 openai provider 的多余自定义模型定义）没有 apiKey → **整个 models.json 自定义模型加载失败** → 所有 zenmux 模型从 catalog 消失 → `resolveGatewayModelSupportsImages` 查不到当前模型 → 返回 false → 附件被丢弃。

### 修复
1. `openclaw.json` 中 `.models.providers.openai.models` 改为 `[]`（gpt-5.4 已是内置模型）
2. `openclaw.json` 中新增 `.models.providers.codex` 带 `apiKey: "oauth-managed"` 占位符，让 merge 后的 models.json 通过校验
3. 配置 `imageModel: openai-codex/gpt-5.4` 用于图片识别
4. 全部 9 个 agent 的 models.json 同步修复

### 教训
- models.json 是**生成文件**，由 `ensureOpenClawModelsJson` 从 openclaw.json 生成，直接改 models.json 无效（Gateway 重启时会覆盖）
- 修复必须改 openclaw.json，不能改 models.json
- 规则 3 违反：前几次修了 models.json 而不是 openclaw.json，导致反复试错、反复重启 Gateway，浪费了 Bruce 大量时间
- 这是 OpenClaw 的 regression bug，应提 issue
