## 20:30-21:08 Zenmux Key 轮换系统性重构（三级保障方案）

### Bruce 的反馈
- 之前的修复全是治标（清冷却、改 lastGood），没从系统层面解决
- 核心需求：`/new` 不换 key；402 时轮换到下一个；全挂时 fallback；冷却过了自动恢复
- key 额度刷新周期约 5 小时

### 根因分析
1. `lastGood` 只对 OAuth 类型有效，对 api_key 完全无用（之前改 lastGood 是白改）
2. `orderProfilesByMode` 按 lastUsed **升序**排（LRU），导致 `/new` 后优先选最久没用的 key（可能已没额度）
3. cooldown bypass 让 402 后不设冷却 → 死 key 永远不被跳过
4. openclaw.json 有 explicit `auth.order`，走不同代码分支，绕过 `orderProfilesByMode`

### 三级保障方案

**Level 1 — Sticky（MRU）**
- `orderProfilesByMode` 排序从 `a.lastUsed - b.lastUsed` 改为 `b.lastUsed - a.lastUsed`
- 文件：auth-profiles-DRuJBw5y.js line 1178
- 效果：`/new` 后优先用上次成功的 key

**Level 2 — Failforward（冷却轮换）**
- 去掉 custom-zenmux-ai 的 cooldown bypass（line 708）
- openclaw.json: billingBackoffHoursByProvider.custom-zenmux-ai = 1（小时）
- 删除 auth.order（让 MRU 排序生效）
- 指数退避：1h→2h→4h，成功后归零

**Level 3 — Fallback（全挂兜底）**
- openclaw.json model.fallbacks: openrouter/anthropic/claude-sonnet-4-20250514 → openai-codex/gpt-5.4
- 原来第一个 fallback 是 custom-zenmux-ai/sonnet（同 provider，key 全挂也没用），已改为 openrouter

|------|------|------|

| auth-profiles-DRuJBw5y.js:708 | 去掉 custom-zenmux-ai bypass | monkey-patch |
| openclaw.json auth.cooldowns | billingBackoffHoursByProvider = 1h | 配置 |
