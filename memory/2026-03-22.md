## Session 延续机制上线

Bruce 发现 session 切换后看不到前面的信息，要求建立延续机制。

### 设计决策

1. 新建 `memory/HANDOFF.md` — 滚动覆盖的接力卡（≤15行），每次对话中有重要进展时更新
2. 每日笔记改为"边聊边写"，不等结束才写
3. 启动协议：先读 HANDOFF.md（极小开销），再读当日笔记，再按需读 MEMORY.md
4. 日常笔记 14 天以上可归档压缩，HANDOFF.md 永远只保留最新状态

### 原则
- 增量写入，不靠一次性 handoff
- 文件永远比会话可靠
- 恢复上下文的成本 < 1KB
